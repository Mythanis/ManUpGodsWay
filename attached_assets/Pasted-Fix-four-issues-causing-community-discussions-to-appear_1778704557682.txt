Fix four issues causing community discussions to appear broken:
Fix 1 — client/src/pages/community.tsx: Fix blank page during auth loading
Find this line (near the bottom of the state declarations):
typescriptconst isLoading = !initialLoaded && isFetching;
Change it to:
typescriptconst isLoading = !initialLoaded;
Without this fix, the page shows completely blank for 3-4 seconds while auth resolves, making users think discussions aren't loading.
Fix 2 — client/src/pages/community.tsx: Fix IntersectionObserver duplicate fetches
Add these 4 refs directly after the existing isFetchingRef and sentinelRef lines:
typescriptconst nextOffsetRef  = useRef(0);
const hasMoreRef     = useRef(true);
const sortByRef      = useRef(sortBy);
const searchQueryRef = useRef(searchQuery);
Add these two effects after the debounce effect:
typescriptuseEffect(() => { sortByRef.current = sortBy; }, [sortBy]);
useEffect(() => { searchQueryRef.current = searchQuery; }, [searchQuery]);
Inside fetchPage, after setHasMore(more) and setNextOffset(next), add:
typescripthasMoreRef.current = more;
nextOffsetRef.current = next;
Inside the reset useEffect, after setNextOffset(0) and setHasMore(true), add:
typescriptnextOffsetRef.current = 0;
hasMoreRef.current = true;
Replace the entire IntersectionObserver useEffect with:
typescriptuseEffect(() => {
  const sentinel = sentinelRef.current;
  if (!sentinel) return;
  const observer = new IntersectionObserver(
    (entries) => {
      if (entries[0].isIntersecting && hasMoreRef.current && !isFetchingRef.current) {
        fetchPage(nextOffsetRef.current, sortByRef.current, searchQueryRef.current);
      }
    },
    { threshold: 0.1 }
  );
  observer.observe(sentinel);
  return () => observer.disconnect();
}, []);
