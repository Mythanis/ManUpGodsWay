Fix 3 — Database: Add indexes on the discussions table
Run these against the production database:
sqlCREATE INDEX IF NOT EXISTS idx_discussions_pinned_created
ON discussions (is_pinned DESC, created_at DESC);

CREATE INDEX IF NOT EXISTS idx_discussions_pinned_likes
ON discussions (is_pinned DESC, likes DESC, created_at DESC);

CREATE INDEX IF NOT EXISTS idx_discussions_pinned_replies
ON discussions (is_pinned DESC, reply_count DESC, created_at DESC);

CREATE INDEX IF NOT EXISTS idx_discussions_user_id
ON discussions (user_id);
These cover all three sort modes (recent/likes/replies). The query currently does a full table scan on every request, causing 1-3 second response times.
