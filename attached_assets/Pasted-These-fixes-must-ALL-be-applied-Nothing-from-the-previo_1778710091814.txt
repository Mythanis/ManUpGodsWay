Fix 1 — client/src/hooks/useTrialAccess.ts — ROOT CAUSE of the 2-3 second discussion loop
staleTime: 0 on subscription-settings causes it to be immediately stale on every mount, triggering a background refetch. During refetch, !settings briefly becomes true, which makes isLoading: true, which hides Community in TrialPageGuard, causing it to unmount and remount. The reset effect fires on every remount and calls fetchPage(0) again — producing the 2-3 second loop.
Make these two changes:
typescript// CHANGE staleTime from 0 to 60 seconds:
const { data: settings } = useQuery<SubscriptionSettings>({
    queryKey: ["/api/subscription-settings"],
    staleTime: 60_000,  // was: staleTime: 0
});

// CHANGE isLoading to not block on settings:
return {
    blocked,
    reason,
    isLoading: !user,  // was: !user || !settings
    isActiveSubscriber,
    isTrialUser,
};
The blocked logic already handles missing settings safely (it defaults to false if settings is undefined). Removing || !settings from isLoading means Community mounts immediately when the user is authenticated, without waiting for subscription-settings to load.
